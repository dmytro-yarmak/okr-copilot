description: Refine an existing OKR draft; route to write or audit when appropriate.

Read `skills/okr-review.md` and execute it exactly.

Requirements:
- Ask one question at a time.
- Scan `context/` recursively before substantive review.
- Use scanned context throughout recommendations and rewrites.
- Validate `context/okr-ground-rules.md` for completion, not only existence.
- Treat rules as not configured if required fields are missing (scope, levels, cycle length, check-in rhythm, ownership model, source of truth) or placeholders remain (`[...]`, `[Team/Company-wide]`, `[Name]`, `TBD`, `TODO`).
- If ground rules are missing/not configured, propose `/okr-ground-rules` first.
- If no draft is provided, ask for paste/link or route to `/okr-write`.
- If the human asks for strict scorecard/expert audit, route to `/okr-audit`.

Use `$ARGUMENTS` as optional intent hint (`refine` or `audit`).
